# Skill 质量规则

读取条件：评审 Skill 想法、起草创建前检查清单、准备交给 `$skill-creator` 前读取。

## 五条底线

1. 一个 Skill 只解决一个明确问题，不做大而全的万能助手。
2. `SKILL.md` 只写模型不知道、但对这个场景很关键的信息。
3. 长内容放到 `references/`，并写清读取条件。
4. 规则要可执行，少写口号，多写判断标准。
5. 好 Skill 要触发准、动作清、输出稳。

## 应该写什么

- 这个 Skill 解决的具体问题。
- 什么时候应该触发。
- 输出应该长什么样。
- 模型容易做错的地方。
- 领域特有的规则、约束、风格和禁忌。

## 不该写什么

- 通用常识，例如 Markdown、Python 的基础概念。
- 大模型本来就知道的泛化建议。
- 没有判断标准的空话，例如“更专业”“更高质量”“深度思考”。
- 本质相同的要求重复出现。

判断每一段是否保留时，问：删掉这段后，模型还会稳定做对吗？如果会，就删。

## description 规则

`description` 使用这个公式：

```text
[做什么] + [输出什么] + [什么时候用] + [用户可能怎么说]
```

合格标准：

- 说明核心能力，不只写领域名。
- 说明输出形式，例如 Markdown、提纲、表格、步骤清单。
- 列出 3 到 5 个典型触发场景。
- 覆盖用户真实会说的话，而不只写概念词。

不要写成：

```yaml
description: 帮助写研究报告
```

要写成能判断触发的描述：

```yaml
description: |
  生成结构化的深度研究报告，支持技术调研、竞品分析和趋势洞察。
  输出为多章节 Markdown，包含结论、论据、案例和来源标注。
  当用户需要研究某个行业、分析某个产品、撰写调研材料、
  比较多个方案，或明确提到“调研”“分析”“趋势”“竞品”时使用。
```

## SKILL.md 推荐内容

- 使用边界
- 任务判断逻辑
- 核心流程
- 输出约束
- references 的读取索引

如果背景说明超过 500 字，优先移到 `references/`。如果大样例超过 3 个，优先移到 `references/examples.md`。如果一个知识块不会每次使用，就不要放在 `SKILL.md`。

## 自由度匹配

创意型 Skill：

- 给方向、原则和风格边界。
- 少给死步骤。
- 多给正反例。

流程型 Skill：

- 规定阶段和检查点。
- 重点约束输出结构和质量标准。
- 不必写死每一步细节。

精确型 Skill：

- 直接给步骤、命令和判断条件。
- 尽量减少模型自行发挥。
- 如果有脚本，优先调用脚本而不是自由生成。

## 必须写清不做什么

- 用户问题与主题无关时不要强行触发。
- 不代替其他 Skill 处理跨领域问题。
- 不在没有明确信号时一次性读取全部 references。
- 不编造数据、案例或来源。
- 如果任务只需要通用能力，不强行套用 Skill。

## 固定分支必须用选项

适用于模式选择、文档类型、输入来源、计划粒度、平台选择、是否开启固定功能。

规则：

- 能枚举合法答案的，就直接给 2 到 5 个选项。
- 只有业务细节、真实案例、禁忌和特殊要求才开放输入。
- `SKILL.md` 中存在固定流程分支时，必须写清可选项。

不要问：

```text
请告诉我你想用哪种模式。
```

要问：

```text
请先选择本次模式：
- 新需求
- 增量需求
```

## 去 AI 味的可执行约束

- 第一段直接进入任务，不铺垫背景。
- 禁止使用“在当今”“随着……发展”“综上所述”“总之”等套话。
- 少用“非常”“极其”“显著地”等空泛副词。
- 不默认套用“首先/其次/最后”的模板。
- 重要观点用事实、案例、数据或具体动作支撑。

## 开发检查清单

- [ ] 这个 Skill 是否只解决一个明确问题？
- [ ] `SKILL.md` 是否只保留高价值、场景特有的信息？
- [ ] `description` 是否写清能力、输出和触发场景？
- [ ] 是否写明不适用场景？
- [ ] 是否明确说明何时读取哪个 `references/` 文件？
- [ ] 固定分支问题是否优先设计成选项式？
- [ ] 是否避免空洞表达和模板化语气？
- [ ] 如果有脚本，是否优先引导模型调用脚本？
- [ ] 是否做到不依赖其他 Skill 目录？

## 最终标准

一个高质量 Skill 应满足四点：

1. 容易被触发。
2. 不容易跑偏。
3. 不浪费上下文。
4. 输出稳定、有信息量、像真人写的。
