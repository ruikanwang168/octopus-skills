# UI 常见模块参考规范 (UI Patterns)

本文件定义了 PC/Web 端及移动端常见业务模块的 PRD 描述规范（CRUD 五要素）。在生成 PRD 时，请参考相应模板，并根据第一阶段识别到的信息进行填充。

---

## 1. 列表管理页描述规范 (List Management)

当识别到“列表页”类型时，PRD 章节应包含以下结构：

### 1.1 页面概要
- **页面定位**：本页面的核心功能和目标用户。
- **入口/路由**：从哪些页面可以进入，如何进入。
- **页面流转**：本页面可以跳转到哪些页面（目标页、传参方式）。

### 1.2 页面布局 (ASCII)
使用 ASCII 绘图展示页面的视觉结构，标注：搜索筛选区、全局操作按钮区、表格/列表区、分页器。

|---|---|---|---|---|---|
| [字段名] | [类型] | [是/否] | [空/具体值] | [功能描述] | [如：切换后自动刷新] |

### 1.4 全局操作按钮区
| 按钮名称 | 类型 | 展示条件 | 点击后行为 | 权限要求 |
|---|---|---|---|---|
| 新建 | 主按钮 | 始终显示 | 跳转新建页/打开新建弹窗 | 创建权限 |
| 批量删除 | 危险按钮 | 选中数据后显示 | 二次确认后删除 | 删除权限 |

## 2. 新建/编辑页描述规范 (Create/Edit Form)

### 2.1 页面概要
- **定位**：创建或修改该业务实体的入口。
- **跳转关系**：完成后自动返回列表页。

### 2.2 表单分区与字段说明
| 字段 | 类型 | 必填 | 校验规则 | 默认值 | 说明 |
|---|---|---|---|---|---|
| [字段名] | [类型] | [是/否] | [正则/范围/唯一性] | [空/具体值] | [业务含义] |

### 2.3 字段联动规则
描述页面上不同字段间的联动（如：选择 A 类型，隐藏 B 字段）。

### 2.4 保存与冲突机制
- **离开确认**：有未保存内容时，关闭页面需二次确认。
- **并发处理**：保存时检测数据是否被他人修改（建议逻辑）。

## 4. 交互反馈模式描述规范 (Interaction Feedback)

为了清晰区分不同的反馈形式，请遵循以下描述标准：

### 4.1 弹窗 (Modal)
- **定义**：居中显示的遮罩层，必须由用户操作（如点击确定/取消）或点击遮罩关闭。
- **适用场景**：二次确认、复杂表单输入、重要信息展示。
- **ASCII 布局**：
  ```text
  +----------------------------+
  | [Modal Title]          [X] |
  +----------------------------+
  |                            |
  |      [Content Area]        |
  |                            |
  +----------------------------+
  |      [Cancel]  [Confirm]   |
  +----------------------------+
  ```

### 4.2 抽屉 (Drawer)
- **定义**：从屏幕边缘（通常是右侧或底部）滑出的面板，不中断主流程但遮挡部分内容。
- **适用场景**：详情展示、侧边设置、辅助任务、不希望丢失主页面上下文的编辑。
- **ASCII 布局**：
  ```text
  +-------------------+-----+
  |                   | [X] |
  |                   |-----|
  |      Main         |     |
  |      Content      | Side|
  |                   |Panel|
  |                   |     |
  +-------------------+-----+
  ```

### 4.3 消息提示 (Message)
- **定义**：页面顶部居中出现的轻量化提示，通常几秒后自动消失。
- **适用场景**：成功操作、一般错误提醒、系统通知。
- **类型标识**：✅ 成功 / ❌ 错误 / ⚠️ 警告 / ℹ️ 提示
- **ASCII 布局**：
  ```text
  [ ✅ 操作成功！ ]
  ```

### 4.4 吐司提示 (Toast)
- **定义**：位于页面角落（Web）或底部（Mobile）的极轻量反馈，不干扰操作，不带操作按钮。
- **适用场景**：状态更新、网络波动提示、自动保存提醒。

## 5. 常见页面类型深度规范 (Page Types)

### 5.1 向导页 (Wizard)
- **定义**：将复杂任务拆分为多个连续步骤的引导式界面。
- **描述要素**：
  - **步骤条**：当前步、完成步、未开始步。
  - **导航逻辑**：上一步、下一步（是否需校验当前页）、取消、提交。
  - **中间态处理**：分步保存草稿逻辑。

### 5.2 结果页 (Result)
- **类型**：成功、警告、失败、404/403、空状态 (Empty)。
- **描述要素**：状态图标、主标题、描述文案、主操作（如：返回列表）、辅助操作（如：查看日志）。

### 5.3 对话页与 AI 交互 (Dialogue & AI)
- **核心组件**：消息列表（含角色区分）、输入框、模型切换。
- **状态描述**：
  - **模型状态**：思考中 (Thought)、流式输出 (Streaming)、完成、报错。
  - **交互操作**：重新生成、停止生成、复制、赞/踩、引用/回复。
- **ASCII 布局**：
  ```text
  +-----------------------------------+
  | [Model Name v]         [Settings] |
  +-----------------------------------+
  | [User]: 帮我分析...               |
  |                                   |
  | [AI]: [Thinking...]               |
  |       依据数据分析，结论如下...   |
  |                                   |
  +-----------------------------------+
  | [ + ] [ Enter prompt...     ] [^] |
  +-----------------------------------+
  ```

## 6. 复杂业务模块规范 (Advanced Modules)

### 6.1 数据看板与可视化 (Dashboard)
- **核心组件**：指标卡 (Metric Cards)、图表 (Charts)、过滤器 (Filters)。
- **描述标准**：
  - **指标卡**：名称、数值、趋势（环比/同比）、单位、时间范围。
  - **图表**：图表类型、数据来源、维度 (X)、度量 (Y)、交互（悬浮提示、下钻）。
- **ASCII 布局**：
  ```text
  +---------+  +---------+  +---------+
  | 总访问量 |  | 转化率  |  | 留存率  |
  | 1,234   |  | 5.6%    |  | 42% ^   |
  +---------+  +---------+  +---------+
  ```

### 6.2 权限管理与 RBAC (Permissions)
- **场景**：用户、角色、权限映射。
- **描述标准**：
  - **角色列表**：角色名称、关联用户数、创建时间。
  - **权限矩阵**：横轴为功能/资源，纵轴为操作（增/删/改/查）。

### 6.3 批处理与导入导出 (Batch Ops)
- **导入流程**：上传文件 -> 解析校验（错误行定位） -> 确认执行 -> 结果概览。
- **导出流程**：条件确认 -> 后台生成 -> 任务中心/直接下载。

## 7. 移动端特有交互 (Mobile Patterns)

### 7.1 底部动作条 (Bottom Sheet)
- **定义**：从屏幕底部向上滑出的半屏面板。
- **适用场景**：单选/多选菜单、关联推荐、快捷设置。

### 7.2 底部导航 (Tab Bar)
- **定义**：位于屏幕底部的顶级导航。
- **描述要素**：图标、文字标签、选中状态、未读气泡/红点。
