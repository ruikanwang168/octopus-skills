# Annotation Type Taxonomy

## 1. 设计目标

`annotationType` 用于表达标注的业务含义和产品交付含义。

它与现有字段的关系如下：

- `kind`：偏工具实现，用于运行时展示和候选生成。
- `dimension`：偏扫描维度，用于描述候选标注来源。
- `topics`：偏主题标签，用于筛选和质量检查。
- `annotationType`：偏产品语义，用于说明该标注在产品设计和交付中的作用。

因此必须保留 `kind`、`dimension`、`topics`，同时新增 `annotationType`。

## 2. 通用标注类型

| annotationType | 名称 | 说明 |
|---|---|---|
| P | 页面级说明标注 | 说明页面名称、目的、角色、核心任务；这是标注类型，不是页面说明文档 |
| E | 入口来源 | 说明用户从哪里进入该页面 |
| C | 组件说明 | 说明页面组件、字段、卡片、模块 |
| A | 操作交互 | 说明点击、输入、上传、提交等操作 |
| J | 页面跳转 | 说明页面之间的跳转关系 |
| S | 状态说明 | 说明加载、成功、失败、审核、禁用等状态 |
| R | 规则说明 | 说明业务规则、判断条件和异常处理 |

## 4. 标注类型使用原则

1. 每条标注建议包含 `annotationType`。
2. 如果无法判断 `annotationType`，默认使用 `C`。
3. 页面级说明标注必须使用 `P`；不要把 `P` 标注描述成独立的页面需求说明文档。
4. 主操作按钮优先使用 `A`。
5. 页面跳转优先使用 `J`。
6. 状态标签、状态流转、异常状态优先使用 `S`。
7. 业务判断、限制条件、校验逻辑优先使用 `R`。
8. AI 输入、处理、输出、兜底相关内容优先使用 `AI`、`HITL`、`FALLBACK`。
