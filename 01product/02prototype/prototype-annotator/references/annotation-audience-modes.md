# 标注受众模式说明

## 1. product-review

默认模式。

适用于：

- 产品评审
- 业务评审
- 客户评审
- 设计评审
- 原型讲解

默认展示：

- 业务含义
- 使用场景
- 交互规则
- 状态与异常
- 页面跳转
- 业务规则
- 待确认

默认隐藏：

- selector
- 源码路径
- store
- state
- mock
- 组件内部实现

## 2. dev-handoff

适用于研发交接。

可以展示：

- 组件路径
- selector
- 数据字段
- store / state
- mock / API
- 技术依赖

但需要放在「研发说明」或「证据」区域中。

## 3. qa-acceptance

适用于测试和验收。

默认展示：

- 前置条件
- 操作步骤
- 预期结果
- 异常路径
- 验收标准

## 4. customer-demo

适用于客户演示和汇报。

默认展示：

- 页面价值
- 用户收益
- 业务流程
- 功能亮点
- AI能力说明
