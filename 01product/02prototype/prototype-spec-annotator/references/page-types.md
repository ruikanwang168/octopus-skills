# Page Types

页面判断分两层：

- 页面类型：决定说明区章节结构
- 页面形态：决定识别重点和额外补充规则

每次至少选：

- 1 个主页面类型
- 1 个主页面形态

## 1. 列表页

主目标：筛选、查看结果、执行单条或批量操作。

建议原型结构：

- 标题区
- 主操作区
- 分类或标签区
- 筛选区
- 结果区
- 分页器

需求说明区固定章节：

1. `【筛选条件】交互规则说明`
2. `【结果区】显示规则说明`
3. `【状态机制】规则说明`
4. `【功能操作】交互规则说明`

至少要覆盖：

- 分类切换
- 筛选项默认值
- 排序和时间格式
- 状态对按钮的影响
- 批量操作的显示条件
- 空状态、失败反馈

## 2. 详情页

主目标：查看对象详情、浏览关联信息、执行主动作。

建议原型结构：

- 标题区
- 状态区
- 基础信息区
- 标签或分栏区
- 操作区
- 更多操作入口

需求说明区固定章节：

1. `【显示隐藏】规则说明`
2. `【基础信息】显示规则说明`
3. `【分栏或标签数据】显示规则说明`
4. `【功能操作】交互规则说明`
5. `【更多操作】规则说明`

至少要覆盖：

- 打开关闭方式
- 关键字段显示方式
- 默认标签
- 标签切换后的加载规则
- 风险操作确认

## 3. 表单页

主目标：输入字段、校验、提交。

建议原型结构：

- 标题区
- 表单主体区
- 字段分组区
- 辅助提示区
- 底部操作区

需求说明区固定章节：

1. `【显示隐藏】规则说明`
2. `【字段输入规则】说明`
3. `【提交规则】说明`

字段输入规则建议输出为表格，至少包含：

- 字段名
- 组件类型
- 规则说明
- 是否必填

至少要覆盖：

- 默认值
- 联动逻辑
- 校验时机
- 错误提示
- 保存草稿和正式提交的差异

## 4. 工具页

主目标：完成一个流程性任务或处理型任务。

建议原型结构：

- 功能说明区
- 输入区
- 结果反馈区
- 操作按钮区
- 状态提示区

需求说明区固定章节：

1. `【功能说明】规则说明`
2. `【输入或校验规则】说明`
3. `【状态说明】规则说明`
4. `【流程说明】规则说明`

至少要覆盖：

- 适用范围和限制
- 输入格式
- 文件类限制
- 状态切换条件
- 中断与恢复路径

## 5. Mixed Pages

如果页面同时带有列表、详情、配置等元素，先判断主任务：

- 主任务是“查结果”，按列表页组织
- 主任务是“看对象”，按详情页组织
- 主任务是“填字段”，按表单页组织
- 主任务是“跑流程”，按工具页组织

次级能力拆到关联页面或次级弹层，不要把多类章节直接混写到一个页面里。

## 6. 页面形态

### 6.1 桌面端页面

适用特征：

- 常规后台、管理台、运营台、业务系统页面
- 以横向空间分区、表格、表单、标签页、抽屉为主

额外应覆盖：

- 侧栏或面包屑导航
- 主副操作的优先级
- 抽屉、弹窗、二级区块的显示隐藏

### 6.2 移动端页面

适用特征：

- 顶部导航、底部固定操作、卡片列表、分段切换、全屏弹层
- 纵向滚动为主，单屏内容密度较高

额外应覆盖：

- 返回路径和关闭路径
- 底部固定按钮的显示条件
- 弱网、加载中、空状态、权限拒绝
- 文案截断、列表折叠、全屏弹层或底部弹层

### 6.3 大屏页面

适用特征：

- 全屏或准全屏布局
- 指标卡、图表、地图、告警、轮播区、时间范围切换
- 数据展示和监控优先于录入

额外应覆盖：

- 数据更新时间或刷新周期
- 时间范围切换
- 图表联动、钻取或高亮关系
- 告警阈值、离线状态、数据缺失
- 自动轮播、自动刷新或全屏展示规则

### 6.4 AIGC 页面

适用特征：

- 提示词输入、参数面板、会话区、生成结果区、引用来源、重试入口
- 以“输入 -> 生成 -> 查看结果 -> 再加工”为主链路

额外应覆盖：

- 输入限制和提交触发方式
- 生成中、停止生成、失败重试
- 部分输出、增量输出或多版本结果
- 引用来源、上下文依赖或附件输入
- 再生成、复制、导出、插入工作流等后续动作

## 7. 其他常见页面形态

以下形态也很常见，不能算遗漏，但当前默认不单独作为一级模板：

### 7.1 平板 / 中间断点页面

处理建议：

- 先判断它更接近移动端还是桌面端
- 额外补分栏折叠、横竖屏切换、侧边区块显示隐藏规则

### 7.2 工作台 / 仪表盘页

处理建议：

- 先判断它更接近列表页、详情页还是工具页
- 额外补卡片取数、待办入口、模块联动和刷新规则

### 7.3 多步骤流程 / 向导页

处理建议：

- 通常按表单页或工具页处理
- 额外补步骤切换、返回上一步、草稿保存、中断恢复规则

### 7.4 二级承载面

包括：

- 抽屉
- 弹窗
- 全屏侧滑层
- 底部弹层

处理建议：

- 不是独立主页面类型，但需要强制补显示隐藏和关闭路径
- 如果二级承载面仅承载轻量确认、简短只读信息或单步输入，可在主页面章节中描述
- 如果二级承载面包含独立列表、表单、筛选、空状态、返回路径、步骤切换或继续进入下一层流程，应升格为“子页面说明”
- 升格后应直接使用真实承载面名称单独成段，例如 `【主数据授权抽屉】规则说明`

### 7.5 文档 / 报告 / 预览 / 打印页

处理建议：

- 通常按详情页或工具页处理
- 额外补版本、导出、打印、水印、只读态规则

### 7.6 地图 / 拓扑 / 画布页

处理建议：

- 通常按工具页或详情页处理
- 额外补视口缩放、节点点击、图例、刷新和异常数据规则
