# 研发计划输出模板

每次生成研发计划包前读取本文件。默认只输出 4 份 Markdown：总计划、前端研发计划、后端研发计划、接口契约。可根据项目术语调整章节名称，但不得增加会突破任务范围的内容。

## 目录

- `00-研发总计划.md`
- `01-前端研发计划.md`
- `02-后端研发计划.md`
- `03-接口契约.md`

## 00-研发总计划.md

不要写选型结论、架构图、实现步骤或方案正文。没有需要单独设计的事项时，明确写“未识别到需要单独产出的技术方案”，并给出判断依据。

#### 现有信息

- 已确认约束：

## 研发顺序与依赖

只描述任务之间的先后和阻塞关系，不估算工时，不分配人员。涉及持久化、查询、保存、发布、删除保护、校验结果承载或数据迁移时，必须写清“数据库设计/变更计划 -> 后端研发 -> 前端联调”的依赖关系。

## 测试重点

## 01-前端研发计划.md

前端研发计划只承载前端任务。不要混入数据库设计、后端实现、接口字段完整定义或测试用例步骤。

每个任务只允许三个正文栏目：`计划`、`前提条件`、`验收标准`。

```md
# 前端研发计划

## FE-01 <任务名称>

### 计划

写明对应需求或页面、影响模块与代码位置、处理方式、主要交付物、所依赖的接口或其他任务。

### 前提条件

- 开始前必须确定、提供或完成的事项。

### 验收标准

- 用户可观察、测试可验证或代码可核对的完成结果。
```

## 02-后端研发计划.md

后端研发计划承载数据库设计/变更计划和后端研发任务。数据库任务不写表名、字段名、字段类型、索引、约束、关系设计或 SQL；若代码中已有相关数据能力，只可在“计划”中引用代码模块或数据职责作为证据。

每个任务只允许三个正文栏目：`计划`、`前提条件`、`验收标准`。

```md
# 后端研发计划

## 数据库设计/变更计划

### DB-01 <任务名称>

#### 计划

写明对应需求、数据职责、复用/修改/新增/删除方式、交付物和依赖。需要详细设计时，把“完成数据库设计并通过确认”作为计划内容或前置依赖，不在此展开。

#### 前提条件

- 开始前必须确定或完成的事项。

#### 验收标准

- 可核对的完成结果。

如果确认不需要数据库变更，写明判断所依据的需求和代码证据，不创建占位任务。

## 后端研发任务

### BE-01 <任务名称>

#### 计划

写明对应需求或业务场景、影响模块与代码位置、处理方式、主要交付物、所依赖的数据库设计、接口决策或其他任务。

#### 前提条件

- 开始前必须确定、提供或完成的事项。涉及持久化、查询、保存、发布、删除保护、校验结果承载或数据迁移时，必须引用对应 `DB-xx`，或写明“数据库设计已确认”。不依赖数据库的后端任务可标明可并行，但仍需列出接口、权限或代码规范等前提。

#### 验收标准

- 接口行为、业务规则、权限、异常或代码结构中可核对的完成结果。
```

## 编号与交叉引用

- 技术方案使用 `TS-xx`，数据库使用 `DB-xx`，前端使用 `FE-xx`，后端使用 `BE-xx`，接口使用 `API-xx`。
- 依赖、风险和待确认项分别使用 `DEP-xx`、`RISK-xx`、`Q-xx`。
- 同一对象在所有文档中保持相同名称和编号。
- 删除任务时同步删除或修正所有交叉引用。
