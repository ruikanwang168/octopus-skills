# 研发计划复核清单

四份文档初稿完成后读取本文件。发现问题时直接修正文档；证据不足时转为待确认项，并标注阻塞范围。

## 输入与证据

- [ ] PRD 或等价需求基线的位置和版本已记录。
- [ ] 原型的页面、路由或截图范围已记录；缺失时已声明局限。
- [ ] 已读取适用范围内的仓库规则和文档约定。
- [ ] 代码可用时，已同时检索前端请求与类型、后端路由与 DTO/VO 或等价结构。
- [ ] 重要代码判断带文件路径、符号名或路由证据。
- [ ] 没有因名称相似就断言已有模块或接口可复用。
- [ ] PRD、原型和代码冲突已记录，没有静默选择其中一方。

## 需求覆盖

- [ ] 每条需求都映射到数据库、前端、后端、接口或“无需研发变更”。
- [ ] 每个原型交互都有对应任务、已有能力证据或待确认项。
- [ ] 删除、替换、兼容和存量数据影响没有遗漏。
- [ ] 权限、异常、空状态、边界状态和回归范围已进入计划或风险。

## 总计划

- [ ] 技术方案清单只列需要产出的方案，没有写方案正文或提前选型。
- [ ] 每项技术方案严格包含方案名称、背景、要解决的问题和现有信息。
- [ ] 研发顺序已说明数据库设计/变更计划、后端研发和前端联调之间的先后或并行关系。
- [ ] 依赖、风险、待确认项和输入冲突已标明阻塞范围。
- [ ] 没有工时估算、人员分配或未经要求的迭代排期。

## 前端研发计划

- [ ] 文件只承载前端研发任务，没有混入数据库设计、后端实现细节或接口完整字段定义。
- [ ] 每项任务只包含计划、前提条件、验收标准三个正文栏目。
- [ ] 计划中包含对应需求或页面、影响范围、处理方式、交付物和依赖。
- [ ] 前提条件是真正阻塞任务开始的接口、设计、素材、交互或环境输入，不是重复计划内容。
- [ ] 验收标准可由用户观察、测试验证或代码核对。

## 后端研发计划

- [ ] 文件包含“数据库设计/变更计划”和“后端研发任务”两个章节。
- [ ] 数据库任务和后端任务都只包含计划、前提条件、验收标准三个正文栏目。
- [ ] 数据库设计/变更计划没有表名、字段名、类型、索引、约束、关系设计或 DDL。
- [ ] 后端计划中包含对应需求或业务场景、影响范围、处理方式、交付物和依赖。
- [ ] 涉及持久化、查询、保存、发布、删除保护、校验结果承载或数据迁移的后端任务，已在前提条件中引用对应 `DB-xx` 或“数据库设计已确认”。
- [ ] 不依赖数据库的后端任务若标注可并行，也列出了接口、权限或代码规范等前提。
- [ ] 验收标准可从接口行为、业务规则、权限、异常或代码结构中核对。

## 接口契约

- [ ] 每个接口都标明复用、修改、新增、替换或待确认。
- [ ] 每个接口都有方法与路径，或项目实际采用的等价接口标识。
- [ ] 请求参数包含位置、名称、类型、必填性、约束和含义。
- [ ] 返回值包含响应包装、字段路径、类型、可空性和含义。
- [ ] 参数与返回类型遵循现有前后端代码；无法确认的内容明确标为待确认。
- [ ] 已有接口同时核对了前端消费方式和后端真实返回，而非只看单侧声明。
- [ ] 新增接口沿用现有路径、命名、响应包装、分页、枚举和空值规范。
- [ ] 接口中的每个前后端改动都有对应 `FE-xx` 和 `BE-xx` 计划，或说明为什么无需改动。

## 跨文档一致性

- [ ] 四个固定文件均已生成。
- [ ] 同一需求、模块、任务和接口在各文档中名称一致。
- [ ] 任务与接口编号唯一，所有交叉引用都能找到目标。
- [ ] 参数名称、类型、必填性和返回字段在所有出现位置一致。
- [ ] 依赖顺序没有循环，或循环依赖已作为风险明确处理。
- [ ] 总计划中的测试重点、发布依赖和回滚关注点与前端、后端计划一致。

## 失败判定

出现任一情况不得交付为完成：

- 默认输出不是 4 份文档，且用户没有明确要求拆分或合并；
- 仍生成 `研发分项计划` 这类合并前端、后端、数据库的默认文件；
- 后端研发计划没有包含数据库设计/变更计划章节；
- 前端研发计划混入数据库设计或后端实现任务；
- 未检索可用代码就断言接口规范或现有接口可复用；
- 技术方案清单包含具体实现方案；
- 数据库设计/变更计划演变成表字段设计；
- 分项任务缺少前提条件或可核对的验收标准；
- 后端持久化类任务没有声明数据库设计或数据库计划依赖；
- 接口缺少参数或返回值定义，且没有明确写“无”或“待确认”；
- 四份文档存在未解释的需求、任务、接口或依赖冲突。
