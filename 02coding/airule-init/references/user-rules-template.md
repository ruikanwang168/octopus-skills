# USER_RULES.md 生成规则

## 文件职责

`USER_RULES.md` 是用户手写规则文件，用来保存项目维护者自己的长期偏好、特殊约束和不希望被技能自动覆盖的项目规则。

这个文件属于用户，不属于 `airule-init` 的增量更新范围。

## 默认模板

仅当目标项目缺少 `USER_RULES.md` 时，初始化模式创建下面的占位内容：

```md
# 用户自定义规则

本文件由项目维护者手动编写，用于记录本项目中特殊的协作偏好、业务约束和长期规则。

`airule-init` 初始化时只创建本文件；后续增量更新不会自动修改或覆盖这里的内容。

## 一、项目特殊约束

待补充。

## 二、个人协作偏好

待补充。

## 三、禁止事项

待补充。
```

## 生成规则

- 只在文件不存在时创建。
- 如果文件已存在，无论内容是否为空，更新模式都不自动修改。
- 可以读取该文件来理解用户偏好，但不能把它当作待同步文件改写。
- 只有用户明确说“修改 USER_RULES.md”或“把这条写入 USER_RULES.md”时，才允许更新。
