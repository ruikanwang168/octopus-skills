# 增量模式规则

## 定位

增量模式只做一件事：同步代码事实和协作入口，不改用户手写规则，也不改用户维护的扫描忽略规则。

它用于项目已经跑了一段时间后，代码结构、技术栈、启动方式、测试方式、协作流程或工具入口发生变化，需要让 AI 规则文件重新跟上项目现状。

## 允许更新的内容

### 1. PROJECT.md

`PROJECT.md` 是增量模式的核心更新对象。

必须重新按 `project-template.md` 的源码检索要求扫描项目，并同步：

- 项目概述
- 核心技术栈
- 主要目录职责
- 架构分层和模块边界
- 启动、构建、测试、格式化、检查命令
- 数据库、外部服务、第三方 SDK 等集成点
- 代码规范与红线

如果旧 `PROJECT.md` 与当前源码不一致，以源码和配置文件为准；无法确认的内容标为“待确认”。

重新扫描源码前，必须先读取并应用 `AI_IGNORE.md`；如果该文件缺失，先按 `ai-ignore-template.md` 创建默认忽略模板，或在写入前临时采用默认忽略范围。

旧内容如何被修正，只写在增量摘要里；`PROJECT.md` 正文只写当前项目事实，不写“旧规则错误”“已移除旧描述”等过程说明。

### 2. AGENTS.md

`AGENTS.md` 只同步协作 SOP，不承载项目技术细节。

可以更新：

- 角色边界和行动前确认规则
- `/AiCoder/` 目录读写边界
- 新功能和老功能修改时的文档同步要求
- 代码输出、安全命令确认、中文沟通等协作规则
- 对 `PROJECT.md`、`USER_RULES.md`、`AI_IGNORE.md` 和工具入口文件的引用

不要把技术栈、目录清单、命令细节大段写入 `AGENTS.md`；这些内容应放在 `PROJECT.md`。

`AGENTS.md` 必须保留规则优先级：`USER_RULES.md` > `AI_IGNORE.md` > `PROJECT.md` > `AGENTS.md` > 工具入口指针文件。

### 3. 工具入口文件

如果目标项目已经存在 `CLAUDE.md`、`.cursor/rules`、`.windsurfrules`、`.cursorrules`，可以将其收敛为 `pointer-files-template.md` 中定义的短指针内容。

如果用户在增量模式中明确新增工具选择，也可以创建对应入口文件。

没有用户明确选择时，不新增不存在的工具入口文件。

### 4. 缺失基础文件

如果缺少以下基础文件，增量模式可以补齐：

- `AGENTS.md`
- `PROJECT.md`
- `USER_RULES.md`
- `AI_IGNORE.md`

其中 `USER_RULES.md` 只在缺失时创建占位，`AI_IGNORE.md` 只在缺失时创建默认忽略模板；如果已经存在，不修改。

## 禁止更新的内容

- 禁止自动修改或重写 `USER_RULES.md`。
- 禁止自动修改或重写 `AI_IGNORE.md`。
- 禁止把用户手写偏好迁移、改写、删减或“优化”。
- 禁止把用户维护的忽略列表迁移、改写、删减或“优化”。
- 禁止因为旧规则看起来不规范就覆盖用户手写内容。
- 禁止修改业务代码、配置文件、数据库脚本或项目源码。
- 禁止生成与源码事实无法对应的技术栈、命令或目录说明。

只有用户明确说“修改 USER_RULES.md”或“把这条写入 USER_RULES.md”时，才允许更新该文件。

只有用户明确说“修改 AI_IGNORE.md”“把这个路径加入忽略”或“更新忽略列表”时，才允许更新该文件。

## 增量工作流

1. 扫描现有规则文件：`AGENTS.md`、`PROJECT.md`、`USER_RULES.md`、`AI_IGNORE.md`、`CLAUDE.md`、`.cursor/rules`、`.windsurfrules`、`.cursorrules`。
2. 读取 `AI_IGNORE.md`；如果缺失，将其列入补齐文件，并在本次扫描中临时采用默认忽略范围。
3. 扫描项目源码和配置，重新建立项目事实，跳过 `AI_IGNORE.md` 声明的忽略路径。
4. 对比旧规则与当前项目事实，标记：
   - 保留：仍然准确且有价值的规则。
   - 更新：与当前代码不一致，需要修正的内容。
   - 补齐：缺失但必须存在的基础规则。
   - 冲突：不同规则文件之间互相矛盾，或规则与源码事实冲突。
   - 待确认：源码无法证明、需要用户确认的内容。
5. 先输出增量摘要，不直接静默覆盖。
6. 用户确认后再写入允许更新的文件。

## 输出摘要要求

写入前的摘要至少包含：

- 将更新的文件
- 将创建的缺失基础文件
- 将创建的 `AiCoder/` 空目录
- `PROJECT.md` 中会同步的代码事实
- `AGENTS.md` 中会调整的协作规则
- 会收敛的工具入口文件
- 明确声明 `USER_RULES.md` 不会被修改
- 明确声明已有 `AI_IGNORE.md` 不会被修改；若缺失，只创建默认忽略模板
- 待确认事项

写入后的摘要只保留：

- 已更新或创建的文件路径
- 已创建的 `AiCoder/` 空目录
- 主要更新点
- 未处理的冲突或待确认事项
